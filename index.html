The thesis

So I built one.

eleven is a pixel-accurate simulator + hot-reload dev loop + RPC bridge for Work Louder devices. Write app.py on your Mac, see it render instantly, deploy the same file to the device when the SDK lands.

Open source · MIT · ~2,100 lines of Python + Swift + a pinch of C.

Clue #3 — the SDK shape

The firmware already
tells us a lot.

ACT · I

Python
on a knob.

A real MicroPython runtime. LVGL. 48 lines for "hello world."

examples/hello.py

Looks familiar?

start() / update() / end() — if you've touched pygame, you know this shape.

The exact same file runs on the device.

lv_root is injected. wlsdk gives you time, rpc, ui, sys. 

import lvgl as lv
import wlsdk

arc = None; label = None

def start():
    global arc, label
    scr = lv_root
    arc = lv.arc(scr)
    arc.set_size(84, 84)
    arc.align(lv.ALIGN.CENTER, 0, -50)
    arc.set_style_arc_color(
        lv.color_hex(0xff8800),
        lv.PART.INDICATOR,
    )
    label = lv.label(scr)
    label.set_text("0%")

def update():
    t = wlsdk.time.get_elapsed()
    pct = int((t * 25) % 101)
    arc.set_value(pct)
    label.set_text("{}%".format(pct))
48 lines total. That's the whole "hello world" — a progress arc that fills 25% per second. This is real MicroPython, not a DSL on top of it.
The GUI layer

LVGL.

The embedded-GUI library inside half the smartwatches you own.

  • C library, MIT-licensed, ~500KB
  • Bindings for MicroPython → full object API
  • Arcs, charts, labels, styles, animations
  • Renders to any framebuffer — SDL on host, SPI LCD on device

ACTUAL FRAMEBUFFER · 100×310 · RENDERED ON MAC

LVGL is the unsexy hero of embedded GUIs. This clock screenshot was rendered by my Mac, not the device — same pixels as the real thing.

ACT · II

The
architecture.

Two layers, four language boundaries, one promise: the device is just another frontend.

The stack
Two boxes. Two layers. The core is MIT and platform-neutral — someone could build a VSCode extension or Linux UI on top without touching core. The device is just another frontend for the same runtime.
The developer experience

Hot reload
+ RPC bridge.

<100ms hot reload

  • FSEvents watches app.py
  • end() tears down LVGL tree
  • importlib.reload()
  • start() rebuilds

It's importlib.reload with GUI-state discipline.

JSON-RPC bridge


# worker.py polls top(1), pushes:
rpc.send("cpu.update", {"value": 42})

# app.py subscribes:
@wlsdk.rpc.register("cpu.update")
def on_cpu(ctx, params):
    target = params["value"]

Host or device — the app doesn't know which.

This is the "demo magic" slide. Hot reload via FSEvents, RPC via JSON. Same mechanism works on host and over USB to the device.
